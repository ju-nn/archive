# AGENTS.md

このリポジトリは、じゅんの個人ポートフォリオサイト / 個人アーカイブです。

Codex がこのリポジトリを編集するときは、このファイルの方針に従ってください。

---

## サイトの目的

このサイトは、じゅんが作ったもの・書いたもの・話したこと・撮ったものをまとめる個人アーカイブです。

目的は以下です。

- 初めて来た人に、何のサイトか分かりやすくする
- じゅんの思想や世界観が自然に伝わるようにする
- note / stand.fm / YouTube / 制作物への導線を自然にする
- 押し売り感なく、支援やアフィリエイトにつながる余地を作る
- 個人サイトらしい静かな雰囲気を保つ

このサイトは、派手な実績アピール用のポートフォリオではありません。

「インターネット上の自室」に近い、静かな個人サイトとして扱ってください。

---

## 大切にする世界観

以下の雰囲気を大切にしてください。

- 静か
- 余白がある
- 個人サイトらしい
- 生活感がある
- やさしい
- 押し売りしない
- 実績より記録に近い
- 働きすぎない暮らし
- セミリタイア
- 生活改善
- 小さな逃げ道
- 写真、文章、音声、制作物がゆるくつながっている感じ

避けたい雰囲気：

- SaaS風
- 広告LP風
- 派手なグラデーション
- 強すぎるCTA
- 過剰なアニメーション
- 実績マウンティング感
- ビジネス感
- 情報商材っぽさ
- アフィリエイトブログっぽさ

主な訪問者は以下です。

- じゅんのnote、stand.fm、YouTube、Xから来た人

## 優先順位

実装や改善を行うときは、以下の順で優先してください。

1. リンク切れや明確な不具合の修正
2. 初回ユーザーの分かりやすさ
3. note / stand.fm / YouTube / 制作物への自然な導線
4. アクセシビリティ
5. SEOの基本対応
6. 個人サイトらしい世界観の強化
7. 支援・アフィリエイト導線
8. 見た目の細かい調整
9. 大きな新機能

迷った場合は、初回ユーザーの分かりやすさと既存の雰囲気を優先してください。

## 1PRの範囲

基本的に、1回の作業は1PRで収まる小さめの変更にしてください。

よい例：

- Hero文言と導線の改善
- リンク切れ修正
- `/about/` ページ追加
- `/uses/` ページ追加
- 外部リンクのアクセシビリティ改善
- meta description の追加
- カテゴリ名の整理

避ける例：

- サイト全体の全面リニューアル
- デザインシステムの作り直し
- 大規模なディレクトリ構成変更
- 既存コンテンツURLの大幅変更
- 複数機能をまとめて大量追加
- 意図のない大規模リファクタリング

---

## 作業範囲と既存変更の扱い

既存の未コミット変更がある場合でも、必要以上に作業範囲を狭めないでください。

作業前には、まず `git status` と必要に応じて `git diff` を確認し、既存変更の内容を把握してください。

目的達成に必要であれば、特定ファイルに限定せず、関連するファイル・画面・コンポーネント・設定を横断的に確認・修正して構いません。

「最小変更」にこだわりすぎず、ユーザー体験・一貫性・完成度を優先してください。

ただし、今回の目的に関係しない大規模リファクタリングや無関係な作り替えは避けてください。

既存変更を勝手に上書き・巻き戻し・削除しないでください。

今回の作業に関係する差分はコミットして構いません。

コミットする場合は、今回の目的に関係する差分だけを含め、無関係な既存変更を混ぜないでください。

コミットメッセージは英語に限定しないでください。個人開発では、日本語のコミットメッセージを使って構いません。

コミットメッセージは、あとから見たときに変更内容が分かりやすいことを優先してください。

`docs:` / `feat:` / `fix:` などの接頭辞は、必要に応じて使って構いません。

コミットメッセージ例：

```text
docs: Codexの作業範囲とコミット方針を明確化
```

コミット前後に、何を変更したか・どの差分を含めたかを簡潔に説明してください。

## 実装前の計画

作業を始める前に、必ず計画を出してください。

計画には以下を含めてください。

- 今回やること
- 今回やらないこと
- 判断理由
- 影響するファイル
- 実行予定のチェック

計画は、1PRで収まる範囲にしてください。

---

## 計画依頼時のルール

ユーザーが「計画」「方針」「提案」「実装前に整理」と依頼した場合は、実装・編集・コミット・PR作成を開始しないでください。

その時点では計画だけを提示し、実装開始前にユーザーの確認を待ってください。

「実装に入ります」「進めます」などの実装開始宣言は、ユーザーが実装を明示的に許可した後に行ってください。

---

## 実装後の報告

実装後は、以下を報告してください。

- 変更内容
- 影響したファイル
- 確認したこと
- 残した課題
- 次のPRでやるとよいこと

リンク修正をした場合は、修正したリンクも報告してください。

## デザイン方針

基本方針：

## トップページの理想構成

トップページは、最終的に以下の構成を目指してください。

1. Hero
   名前、キャッチコピー、短い説明

2. はじめての方へ
   このサイトが何かを説明する短いセクション

3. まず見るなら
   初回ユーザー向けのおすすめ導線

4. つくってきたもの
   制作物、記事、音声、動画、写真の一覧

5. つかっているもの
   `/uses/` への自然な導線

6. プロフィール抜粋
   `/about/` への導線

7. フッター
   外部リンク、支援リンク、アフィリエイト表記など

一度に全部実装する必要はありません。

推奨カテゴリ：

- すべて

- 文章
- 音声
- 動画
- 写真
- つかっているもの

既存の `note` / `stand.fm` / `YouTube` カテゴリがある場合は、急に削除せず、影響を確認してから段階的に整理してください。

## 外部リンク方針

外部リンクには以下を設定してください。

```html
target="_blank" rel="noopener noreferrer"
```

また、アイコンリンクには必ず `aria-label` を付けてください。

例：

```html
<a href="..." target="_blank" rel="noopener noreferrer" aria-label="noteを開く">
  ...
</a>
```

対象：

- note
- stand.fm
- YouTube
- X
- GitHub
- その他外部サイト

---

## SEO方針

SEOは過剰にやらず、基本を整えてください。

優先すること：

- ページごとの `title`
- ページごとの `meta description`
- OGP設定
- 画像の `alt`
- 適切な `h1` / `h2`
- リンク切れ修正
- サイトマップがある場合は更新

トップページの meta description 例：

```text
じゅんの個人アーカイブ。働きすぎない暮らし、セミリタイア、生活改善、制作物、写真、note、stand.fm、YouTubeをまとめています。
```

## `/about/` ページ方針

`/about/` ページを追加・編集するときは、以下の方向性にしてください。

目的：

- じゅんが何者か分かる
- 価値観や世界観が伝わる
- 自己PR感を強くしすぎない
- 各外部メディアへの導線を置く

プロフィール文の例：

```text
じゅん / ゆるセミリタイア

会社員として働きながら、働きすぎない暮らしと小さな逃げ道を探しています。
生活改善、セミリタイア、写真、ゲーム、植物、猫、カメラなどに関心があります。

このサイトでは、作ったツール、書いた文章、録った音声、撮った写真をまとめています。
派手な実績集というより、自分の生活と考え方を置いておくための個人アーカイブです。
```

## `/uses/` ページ方針

`/uses/` は、じゅんが実際に使っているものをまとめるページです。

目的：

- 生活や制作の道具を記録する
- 読者の参考になる
- 将来的にアフィリエイト導線にもできる
- ただし押し売り感は出さない

説明文の例：

```text
日々の生活、写真、動画、開発、セミリタイア準備で実際に使っているものをまとめています。
無理におすすめするというより、自分の暮らしに残った道具の記録です。
```

カテゴリ例：

- 生活
- ガジェット
- カメラ
- 開発・AI
- 読書・学習
- 猫・植物

カード項目：

- 商品名
- 用途
- ひとこと感想
- 関連記事リンク
- 購入リンクまたは詳細リンク

アフィリエイト表記例：

```text
一部リンクにはアフィリエイトを含む場合があります。実際に使ってよかったものだけを、暮らしの記録として載せています。
```

---

## 支援・収益化導線

支援やアフィリエイト導線は、小さく自然に置いてください。

よい例：

```text
このサイトや制作物が役に立ったら、応援してもらえるとうれしいです。
```

避けること：

- 強い購入誘導
- 過剰な広告配置
- 記事の途中に何度も出す
- 収益化を前面に出す
- アフィリエイトリンクを隠す

支援リンクやアフィリエイトリンクは、設定ファイルやデータファイルでON/OFFしやすい形にしてください。

## 制作物ページ方針

制作物ページでは、単なるリンク集ではなく、背景が伝わるようにしてください。

入れるとよい項目：

- これは何か
- なぜ作ったか
- 誰向けか
- 主な機能
- 使った技術
- 関連記事
- 今後やりたいこと

テンプレート例：

```markdown
# 作品名

## これは何？

短い説明。

## 作った理由

なぜ作ったか。

## 向いている人

- xxx
- xxx

## 主な機能

- xxx
- xxx

## 関連リンク

- アプリを見る
- 開発メモを読む
```

## リンク切れ対応

制作物や外部リンクを変更するときは、リンク切れを確認してください。

特に確認するもの：

- 制作物カード
- note
- stand.fm
- YouTube
- X
- GitHub
- `/about/`
- `/uses/`

存在しないページへのリンクは避けてください。

未公開コンテンツの場合は、以下のいずれかにしてください。

- カードを非表示にする
- 詳細ページを作る
- Coming soon 表示にする
- クリック不可にする

---

## アクセシビリティ方針

最低限、以下を守ってください。

- アイコンリンクには `aria-label` を付ける
- 画像には適切な `alt` を付ける
- 見出し順序を大きく崩さない
- キーボード操作で主要リンクに移動できる
- 色だけで意味を伝えない
- 文字サイズを小さくしすぎない

---

## 禁止事項

以下は避けてください。

- 事前計画なしで大きな変更をする
- 既存の雰囲気を壊す
- 派手なLP風にする
- 全面リニューアルする
- 不要な依存関係を追加する
- 使われていない大規模なライブラリを入れる
- 既存URLを理由なく変える
- 外部リンクを同一タブで強制遷移させる
- アフィリエイト表記なしで購入リンクを置く
- 過度なSEO文言を詰め込む

---

## チェック項目

作業後、可能な範囲で以下を確認してください。

- ビルドが通る
- Lintが通る
- 主要ページが表示される
- モバイル表示が崩れていない
- 外部リンクが適切に開く
- 404リンクが増えていない
- meta description が入っている
- 画像に alt がある
- 既存の雰囲気が壊れていない

---

## 判断に迷ったとき

判断に迷った場合は、以下を優先してください。

1. 初めて来た人が迷わないこと
2. 個人サイトらしい静けさを保つこと
3. 大きく作り替えず、小さく改善すること
4. 押し売り感を出さないこと
5. じゅんの生活・思想・制作物が自然につながること
